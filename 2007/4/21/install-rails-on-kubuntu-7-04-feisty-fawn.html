Install Rails on Kubuntu 7.04 [Feisty Fawn]

April 21st, 2007

Kubuntu 7.04 was just released this week. I documented my setup of Ruby on Rails.

Install Ruby and Dependencies

  1. K Menu > System > Adept Manager

  2. Adept > Manage Repositories

  3. Search for “ruby” [no quotes]

  4. Add the following to your install queue by clicking each one chosing “Request Install”:

    • ri
    • rake
    • rdoc
    • ruby1.8-dev
    • rubygems

    The irb and ruby packages should already be installed.

  5. Search for “build-essentials”. Add it to your install queue.

    • build-essentials - Required for Mongrel gem install later

Database

Still in Adept:

  1. Search for “sqlite”.

  2. Add these packages to your queue:

    • libsqlite3-ruby
    • sqlite3

    You may use the same method above to install other database technologies. Just find both files for your database product—the Ruby library file [eg, libmysql-ruby] and the database app/server itself [eg, mysql-server].

  3. Click “Apply Changes” in the top toolbar. All packages will be installed.

Update Gems

Close Adept. This will be the console portion of our install.

  1. K Menu > System > Konsole

  2. Update RubyGems



$ sudo gem update --system

Install Rails



$ sudo gem install rails --include-dependencies

The –include-dependencies switch causes an unattended install. It automatically answers “yes” in the background to any questions about dependencies. It saves you from having to click “Y” several times.

Web Server

Install Mongrel



$ sudo gem install mongrel

You will be asked which version to install. Choose the latest “ruby” verion [not mswin32]. Answer “Y” to any questions about dependencies.

Verify Install

Verify Apps and Frameworks

1
2
3

$ ruby --version
$ rails --version
$ sqlite3 --version

You should see a version returned for each command above.

Verify Gems



$ gem list

You should see the following gems:

  • actionmailer
  • actionpack
  • actionwebservice
  • activerecord
  • activesupport
  • cgi_multipart_eof_fix
  • daemons
  • fastthread
  • gem_plugin
  • mongrel
  • rails
  • rake
  • sources

Verify Web Server

Change to the directory of a Rails app and start Mongrel.

1
2

$ cd /home/russ/dev/rails/rb.com
$ mongrel_rails start

If you do not have a Rails app, you can create one by typing ”rails test” in the directory where you would like to keep you Rails development apps. This creates a skeletal Rails app called “test”.

Browse http://localhost:3000 in a browser. Your Rails app will display.

Exit Mongrel at the Konsole with Ctrl-C.
