Five Problems in a Single Tag

April 6th, 2007

I found this grossly problematic tag checked into a source-control system recently. I had to save it because I’ve never seen this many problems exist in a single HTML tag.



<font style="font-size: 8pt; vertical-align: middle; color: #666666;">

  1. The <font> tag has been deprecated since the fall of the HTML 4 spec about 8 years ago because it is considered inline style. It is regarded as a mistake by those who added it to early specs. It’s use is highly frowned upon in the web development community and I’m surprised this news has not caught up to this “senior” developer.

    Because it has a style attribute, we have…

  2. Inline style embedded in inline style. Very creative. I’ve never seen that before in my decade of web experience.

  3. “vertical-align: middle” makes child elements within a block-level element align to the middle. It appears as if the developer thinks this style property will align the element within the parent element - or perhaps within the line that it’s in. Impossible. <font> is an inline element, not block. It can not have elements vertically aligned within it.

  4. Middle vertical alignment is the default for every block level element since the browser was invented. It’s an ambiguous style. Same as valign=”middle” and align=”left”. Pure evil.

  5. A non-abbreviated color value. It shouldn’t be there to begin with, but if you’re going to use it, at least use the shortcut, #666;
