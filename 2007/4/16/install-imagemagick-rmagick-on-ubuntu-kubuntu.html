Install ImageMagick & RMagick on Ubuntu / Kubuntu

April 16th, 2007

Installing ImageMagick and RMagick on Kubuntu wasn’t so straight-forward. After following the official ImageMagick instructions, when installing the RMagick gem, there were several errors regarding missing GhostScript fonts.

I discovered that ImageMagick’s “configure” script was looking in the wrong place. The script appears to look in the following locations:

1
2
3
4
5

/usr/share/fonts/default/Type1/
/usr/share/ghostscript/fonts/
/usr/share/fonts/ghostscript/
/sw/share/ghostscript/fonts
/System/Library/Frameworks/Ghostscript.framework/Resources/fonts

But, Kubuntu’s fonts reside in:



/usr/share/fonts/type1/gsfonts

It was a matter of passing ImageMagick’s configure script an additional parameter with the correct font directory, with-gs-font-dir=/usr/share/fonts/type1/gsfonts.

1
2

./configure --disable-static --with-modules --without-perl --without-magick-plus-plus \
--with-quantum-depth=8 --with-gs-font-dir=/usr/share/fonts/type1/gsfonts

Now, you’ll be able to proceed with compiling ImageMagick…

1
2

make
sudo make install

…and installing the gem:



sudo gem install rmagick

If successful, you’ll see this:

1
2
3

Bulk updating Gem source index for: http://gems.rubyforge.org
Building native extensions.  This could take a while...
Successfully installed rmagick-X.Y.Z
