This section describes how this site was made, the tools used, and the technologies behind it.

Design

Graphic

This site was hand-crafted using the finest two-dimensional graphics application in the world, Adobe Photoshop.

Color

Warm and cozy - nothing jarring and boisterous. I knew I wanted earth tones this time because Mother Nature is often the best art-school professor and, frankly, my last site was a kitschy Christmas lawn ornament, built 7 years ago, prior to my research into color science. During a recent Doo-Wop–themed design for a Harley-Davidson dealer, I happened upon and fell in love with this bistre, powder blue, saffron combination. It hearkens back to 40’s and 50’s-era design, for which, I have a new-found admiration. I desaturated it to relax it and shifted the colors somewhat to bring an ever so small amount of green into the browns—slightly olive if you stare long enough, emulating the primary wall color in my home [which those ever-clever, paint-naming folks tagged, “Khaki Shadow”].

Markup and Stylesheets

Hypertext is hand-authored to the XHTML 1.0 Strict standard. Cascading Style Sheets were also hand-authored and contain a combination of 1.0, 2.0, and 3.0 properties.

Photography

All photography throughout the site is mine.

Typography

  • Body Font - Trebuchet - 1.6em leading
  • Headings - Georgia, Small-Caps. I was using fascinating technique called, Flash Satay, to display headings in my logo font, Minister, but performance suffered too much with a lot of headings on the page.
  • Logo - A gorgeous Adobe typeface called “Minister” that I finally found after hours of scouring the Adobe Font Folio, OpenType Edition. I wanted classy but subtly informal. The small caps, Melior x-height, and the below-the-baseline stem on the “R” are all characteristics of my own handwriting. Minister’s oblique stresses and concave serifs keep it playful and less stuffy. Perfect.
  • Nav Bar - Futura Bold - A timeless classic!
  • Sidebar Headings - Futura Heavy
  • Type on Photos - Bickham Script - Ligatures, swash, and ornaments active

Programming Language and Framework

Ruby on Rails

Born and raise Microsoft Windows, I initially spent about three months, nights and weekends, writing this entire site from scratch in C# on ASP.NET. I had the journal completely finished [I refuse to use the term “blog”], as well as the user management system, authentication and role-based permissioning system, and the core workings of a photo gallery. Somewhere in the midst of that process, Ruby on Rails caught my fascination and I committed myself to learning it. I decided that I would rewrite my site on Rails to learn the technology. I bought every known textbook on Ruby and Rails, built a Linux box [Kubuntu], and immersed myself for another several months in this innovative web development framework. What you are looking at is the culmination of those efforts.

Development Tools

Eclipse

I’m using the Eclipse IDE with the Subclipse and RadRails plugins.

Revision Control System

Subversion

All source code is held in Subversion running on Windows 2003 Server.

Database

SQLite

I bounced back and forth between MySQL and SQLite during development. I decided that I would launch the site with a SQLite back-end, given the site’s very low load. I love the portability and elegant simplicity of SQLite. It “just works” and it’s fast. Since it is a single file and resides within my Eclipse project, I can check it in and out of Subversion as I bounce between my Windows and Linux workstations. If my traffic suddenly spikes to thousands of concurrent connections, I’ll explore MySQL or PostgreS.

Operating System

Kubuntu

Of course, I had to run the Linux OS that everyone is talking about! <kidding> After a lot of research, the Ubuntu family seemed like the wise choice. I went with Kubuntu over Ubuntu because I like the KDE user-interface better than Gnome. Gnome seems to have sacrificed power for simplicity. I prefer power and, apparently, so does Linux creator, Linus Torvalds.

I should mention that my early dabblings in Ruby on Rails were on Windows XP, against my better judgment. Experience has repeatedly indicated to me that software runs better in its native environment. At some point, I hit a few obstacles on Windows XP that would not have existed on Linux. Without getting into details here, let’s just say that I am now successfully developing and running Rails in both environments, but things were far more seamless [less troublesome] on the Linux side, as expected. Plus, everything performs better, from development tools to frameworks to web servers to database technologies. Do yourself a favor. Run software in its native environment.

Web Server

nginx / Mongrel Cluster

Pronounced “engine x”, nginx is the latest lightening-fast, rock-solid HTTP server from Russia. Performance tests show that nginx is 50% faster than Lighttpd and Lighttpd is 250% faster than Apache. Unlike the two aforementioned products, it has a very small memory footprint and no leakage problems. Apache has been shown to leak 20MB per day with certain Rails apps. Lighttpd has known bugs in its mod_proxy plugin that cause repeated 500 errors, and the developers don’t appear to be attending to it any time soon. The Mongrel developer actually advises against Lighttpd’s usage. Given all these facts, nginx is my front-line reverse proxy serving all requests for static content, for example, images, JavaScript, and CSS. Ruby on Rails execution is handled by a cluster of three Mongrel instances, with nginx distributing the load. I am thoroughly impressed with the speed and stability of this setup.
